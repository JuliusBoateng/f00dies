Development Plan

Description: The project has two searches, a recipe search and a restaurant search. For the restaurant searches,
the user enters general food dishes such as coffee, hamburgers, chicken, etc. The search matches the
item to menu items listed on campus restaurant menus. The restaurant along with its location, name,
and hours are returned as well as the menu item with its price, calories, allergy, and “student rating”.
The “student rating” will be calculated through an algorithm that the group will develop through user user
data in the form of a survey. The recipe search will be processed by creating an entry that matches the general
dish being searched with a link to the recipe.

Usefulness: The project will be useful because the campus dining app does not detail which location sells which product,
but only gives a general overview of the restaurant. Similar apps such as Yelp only provides restaurants known for selling
foods that match the specific type of food searched but does not do a good job displaying nondescript restaurants that sell
the food being searched. Another similar app is supercook, which is an application that lets the user input ingredients to
search for recipes. Our project is different because the user can get information on physical restaurants that sells the food
item they want as well as provide recipes.

Realness: The data for our project will come primarily from Notre Dame Dining. Notre Dame probably has a database that lists all
of the different foods that they sell along with their restaurants. The recipe information can be provided through a recipe websites.

Functionality: Basic function: Insert menu items and restaurants into database with data from ND dining. Output the appropriate restaurant
add menu items if the user food search matches the general category of a menu item. For the recipe search, insert recipe links into database.
Output link if the user’s general food search category matches a recipe category.

Advanced function: 1.) Getting recipes for dishes based on user constraints such as calories and allergens.
2.) Constantly getting user data by attaching a survey to our site and using that information to rate the menu items
and categorize the search results.
