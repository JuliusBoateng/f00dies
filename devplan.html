R1 = Recipe(name, dish_simple_name, link, calories, al1, al2, al3, al4, al5, al6, al7, al8)
R2 = Dishes(simple_name, category)
R3 = Menu_Items(name, rest_name, rest_location, dish_simple_name, category, price, calories, rating, al1, al2, al3, al4, al5, al6, al7, al8)
R4 = Restaurants(name, location, hours)

S1 = {name -> dish_simple_name, link, calories, al1, al2, al3, al4, al5, al6, al7, al8}
S2 = {simple_name -> category}
S3 = {name, rest_name, rest_location -> dish_simple_name, category, price, calories, rating, al1, al2, al3, al4, al5, al6, al7, al8}
S4 = {name, location -> hours}

Software Tools

Angular, HTML, CSS, Javascript, Java, C++, Python, MySQL

HTML/CSS/Javascript/Angular will be used on the front-end, while one/some of the other listed language(s) will be used for the back-end/middle-ware logic (e.g. advanced functions). Of course, we will be using MySQL for our database.

Data

The restaurant data will come from either the http://nutrition.nd.edu/1 website or from a spreadsheet or raw file from dining@nd.edu. Spoonacular API will be used to fill the recipe database. The data for our restaurant results ordering algorithm will come from ND students from our various classes.
